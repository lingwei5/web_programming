根目录

根目录是文件系统的起点，通常用 / 表示。在Web应用中，它是服务器上网站内容的最顶层目录。

  • 绝对路径总是从根目录开始
  • Web服务器的根目录通常映射到域名
  • 示例：/var/www/html/ 或 Windows 上的 C:\xampp\htdocs\

相对目录

相对目录是相对于当前工作目录的路径。在Web开发中，常用于引用资源文件。

  • styles.css - 当前目录下的文件
  • ../images/logo.png - 上一级目录中的文件
  • ./js/app.js - 当前目录下的文件（显式表示）

URL路径

URL路径表示Web资源在服务器上的位置，由浏览器解析并发送到服务器。

  • https://example.com/blog/article.html
  • 协议: https
  • 域名: example.com
  • 路径: /blog/article.html

路径解析过程

当浏览器请求一个URL时：

  1. 解析协议（HTTP/HTTPS）确定连接方式
  2. 通过DNS查找域名对应的服务器IP地址
  3. 将路径部分发送到Web服务器（如：/projects/website/public/index.html）
  4. Web服务器在根目录（如 /var/www）下查找请求的路径
  5. 服务器返回找到的资源或生成动态内容

根目录是起点，相对路径基于当前位置，URL路径是浏览器与服务器的通信方式
